Section 1: The Workspace (PARA)

Your agent's brain isn't in the cloud; it's on your disk. If it isn't in a file, it doesn't exist. Period. I use the PARA method to keep my agent's mind sharp and organized. If your directory structure is a mess, your agent's logic will be too.

The Golden Rule: The filesystem is first-class memory. We don't rely on "infinite context." We rely on high-density, structured retrieval.

Path Function Retention
/logic/ Core scripts (heartbeat, sync, backoff) Permanent
/memory/ Daily logs (YYYY-MM-DD.md) 30 Days
/memory/long-term/ Curated insights (MEMORY.md) Permanent
/projects/ Active work divided by status Variable
/resources/ Docs, research, and vector indices Permanent

Section 2: The Logic (SOUL & PIDs)

How do you give an AI a personality that doesn't suck? You write it down in SOUL.md. This isn't a "system prompt" you send once; it's the agent's constitution. It's how it decides what's important when you're not looking.

Vector RAG (The Massive Librarian)

When your /resources/ folder gets too big for a simple grep, you need a vector database. Here's a quick Python script to keep your local memory indexed.

import os
import chromadb
from sentence_transformers import SentenceTransformer

# SETUP
client = chromadb.PersistentClient(path="./resources/vectors")
collection = client.get_or_create_collection(name="sovereign_memory")
model = SentenceTransformer('all-MiniLM-L6-v2')

def index_resources():
    resource_path = "./resources"
    for root, dirs, files in os.walk(resource_path):
        for file in files:
            if file.endswith(".md"):
                path = os.path.join(root, file)
                with open(path, 'r') as f:
                    content = f.read()
                    # Basic chunking
                    chunks = [content[i:i+500] for i in range(0, len(content), 450)]
                    for i, chunk in enumerate(chunks):
                        embedding = model.encode(chunk).tolist()
                        collection.add(
                            documents=[chunk],
                            embeddings=[embedding],
                            ids=[f"{path}_{i}"],
                            metadatas=[{"path": path}]
                        )

if __name__ == "__main__":
    index_resources()

Section 3: The Swarm (Sub-agents)

Don't build a "God Agent." It's a single point of failure and it gets confused. Instead, be the CEO. Spawn sub-agents for specific tasks, harvest their output, and then kill them. It's clean, efficient, and way more powerful.

"Orchestration over Monoliths. Always."

Section 4: The Pulse (Heartbeat & Sync)

This is where the magic happens. A Sovereign Agent doesn't wait for you. It checks in on itself using a cron job we call the Heartbeat. Every 15 minutes, it wakes up, checks security, checks for tasks, and syncs its state to Git.

The Heartbeat Script (heartbeat.sh)

#!/bin/bash
# heartbeat.sh - The Sovereign Pulse
WORKSPACE="/Users/ozagent/.openclaw/workspace"
LOG_FILE="$WORKSPACE/memory/heartbeats/$(date +'%Y-%m-%d').log"

echo "[$(date +'%T')] HEARTBEAT START" >> "$LOG_FILE"

# 1. SECURITY AUDIT (Look for accidental key exposure)
GREP_KEYS=$(grep -rE "AI_API_KEY|TOKEN" "$WORKSPACE" --exclude-dir=".git")
if [ -n "$GREP_KEYS" ]; then
    echo "ALERT: Exposed keys detected!" >> "$LOG_FILE"
fi

# 2. GIT PERSISTENCE (Auto-save progress)
cd "$WORKSPACE"
if [[ $(git status --porcelain) ]]; then
    git add .
    git commit -m "Auto-sync: $(date +'%Y-%m-%d %H:%M:%S')"
    git push origin main
fi

# 3. HEALTH CHECK
openclaw gateway status | grep -q "running" || openclaw gateway restart

echo "HEARTBEAT COMPLETE" >> "$LOG_FILE"
