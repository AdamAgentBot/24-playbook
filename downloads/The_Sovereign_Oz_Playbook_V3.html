3. ORCHESTRATION & ISOLATION

The "God Agent" is a myth. Complexity kills focus. The Sovereign Engine scales by spawning specialized, ephemeral sub-agents to handle narrow tasks. This section details the protocol for secure, recursive delegation.

TECHNICAL SPEC The 20-Step Handoff Protocol

A successful handoff is a high-fidelity context transfer. If a sub-agent fails, the error usually lies in Step 1 through 5.

  1. 01. Bottleneck Identification: Main agent determines if the task requires > 3 tool calls or complex file editing.
  2. 02. Context Pruning: Extracting only the relevant snippets from PARA to avoid token bloat.
  3. 03. Persona Mapping: Assigning a specific role (e.g., [Dev Builder] or [Scout Researcher]).
  4. 04. Tool Authorization: Granting temporary, limited tool permissions to the sub-agent.
  5. 05. Isolation Spawning: Initializing the sub-agent in a dedicated ephemeral subdirectory.
  6. 06. State Sync: Sub-agent reads the task.json and initializes its local memory.
  7. ... Remaining steps involve execution loops, QA verification, and distillation.

5. RESILIENCE & EVOLUTION

True sovereignty is the ability to withstand failure. This final section outlines the defensive layers that prevent the engine from collapsing during external API outages or model hallucinations.

DEFENSE API Rate Limit Armor

If you treat APIs like a faucet, you will be cut off. Treat them like a limited resource.

A.

Exponential Backoff: Hardcoded wait times of 10s, 60s, and 5m upon detecting a 429 error code. Retries are restricted to a maximum of 3 before the task is locked.

B.

The Circuit Breaker: If a tool call fails 3 times consecutively with the same error, the task enters `FAILED_LOCKED` status. The engine stops token consumption and waits for a human CEO override.
